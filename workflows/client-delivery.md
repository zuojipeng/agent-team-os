# Workflow: Client Delivery Mode

Use when taking external work.

## Principles

- Confirm scope in writing.
- Make acceptance criteria testable.
- Separate discovery, build, revision, and handoff.
- Avoid unpaid scope expansion.
- Keep demo and deployment evidence.

## Stages

1. Intake: goals, constraints, stakeholders, budget, timeline.
2. Proposal: scope, non-goals, milestones, risks, acceptance.
3. Design: UE spec and technical approach.
4. Build: small deliverable slices.
5. Review: client feedback mapped to scope.
6. Test: acceptance checklist.
7. Handoff: docs, credentials, deploy notes, maintenance options.

## Required Artifacts

- Client brief
- Scope and non-goals
- Milestone plan
- Change request log
- Test report
- Handoff document
