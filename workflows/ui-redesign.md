# Workflow: UI Redesign Mode

Use when improving interface quality, not adding backend capability.

## Sequence

1. UEAgent: identify product shape and primary workflow.
2. UEAgent: audit current hierarchy, density, state coverage, accessibility.
3. Product Agent: confirm what should not change.
4. Architecture Agent: identify frontend boundaries and refactor risks.
5. Engineering Agent: implement one screen or component group.
6. Test Agent: browser screenshots, responsive checks, E2E path.
7. Operator Agent: before/after notes.

## Review Checklist

- Does the primary action stand out?
- Are states visible?
- Does the layout support repeated work?
- Does text fit on mobile and desktop?
- Does color encode meaning?
- Did the redesign avoid decorative complexity?
