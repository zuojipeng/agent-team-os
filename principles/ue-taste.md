# UE Taste Principles

## Experience Before Components

Map:

```text
User intent
  -> Needed information
  -> Decision
  -> Action
  -> Feedback
  -> Recovery
```

Then choose layout and components.

## One Primary Path

Each screen needs:
- One dominant goal
- One primary action
- Clear secondary actions
- Visible status
- Recoverable failure state

If many actions have equal visual weight, the design has not made a decision.

## Professional Interface Rules

Use hierarchy, not decoration:
- Position
- Spacing
- Typography scale
- Contrast
- Grouping
- State indicators
- Progressive disclosure

Avoid:
- Nested cards
- Decorative dashboards
- Instruction text that compensates for unclear UI
- One-note color palettes
- Oversized hero type inside work surfaces

## State Design Is Mandatory

Define:
- Empty
- Loading
- Success
- Error
- Partial data
- Disabled
- Retry/recovery
- Mobile behavior
