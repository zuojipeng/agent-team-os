# Agent Review Rubric

Every Agent must answer:
- What problem did this solve?
- What changed?
- What did not change?
- What evidence proves it works?
- What risk remains?
- Who owns follow-up?

## Product

PASS:
- User job is concrete.
- Success is observable.
- Scope and non-goals are clear.
- Acceptance criteria can become tests.

## UE

PASS:
- Primary path is clear.
- States are specified.
- Responsive behavior exists.
- UI matches product shape.

## Architecture

PASS:
- Smallest sufficient design.
- Domain boundaries named.
- Data contracts explicit.
- Abstractions justified.

## Engineering

PASS:
- Scoped implementation.
- Existing patterns respected.
- Types and errors handled.
- Tests match risk.

## Code Review

PASS:
- Findings first.
- Severity clear.
- File/line references.
- No P0/P1 issue.

## Test

PASS:
- Tests map to acceptance criteria.
- Core path evidence exists.
- Residual risk stated.

## DevOps

PASS:
- Deploy steps, config, smoke, rollback, and monitoring are explicit.
