# Agent Team Maturity Model

Use this to judge whether Agent Team OS is becoming a real delivery organization.

## Level 1: Prompt Pack

Traits:
- roles are named
- workflows are written
- human still drives most sequencing

Risk:
- process is easy to ignore

## Level 2: Repeatable Delivery

Traits:
- projects have pointers to Team OS
- playbooks and templates are used
- reports and tests are produced
- code review is documented

Risk:
- quality still depends on human vigilance

## Level 3: Supervised Team

Traits:
- roles review each other
- failures route to owners
- scorecards track project health
- ADRs capture architecture decisions
- retrospectives update memory

Risk:
- communication overhead can grow

## Level 4: Autonomous Delivery Unit

Traits:
- team can take over projects
- choose next slices
- validate and report evidence
- improve Team OS after each case
- run across Codex, Claude, Cursor, OpenClaw

Risk:
- human must still guard strategic direction and commitments

## Level 5: Productized Agent Organization

Traits:
- case studies demonstrate outcomes
- onboarding is one command
- dashboards show team/project health
- clients can understand deliverables
- reusable methods become a sellable asset

Risk:
- methodology can become stale without real project pressure
