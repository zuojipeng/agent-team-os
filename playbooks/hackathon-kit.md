# Playbook: Hackathon Kit

Use when building for a timed competition.

## Principle

Win with a sharp demo, not a broad system.

## 48h Plan

### 0-4h: Shape

- One sentence problem.
- One target user.
- One demo promise.
- One judging hook.
- One technical risk.

### 4-28h: Build

- Build only the demo path.
- Use mocks for unstable dependencies.
- Keep data local unless online collaboration is central.
- Commit at stable checkpoints.

### 28-40h: Polish

- UEAgent improves hierarchy, copy, loading, empty, error.
- Operator Agent writes pitch and demo script.
- Test Agent locks smoke path.

### 40-48h: Freeze

- No new features.
- Fix demo bugs only.
- Prepare fallback recording.
- Write setup instructions.

## Output

- Demo app
- Pitch script
- Setup notes
- Known limitation list
- Fallback video or screenshots
