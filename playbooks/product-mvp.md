# Playbook: Product MVP

Use when starting a product from an idea.

## Sequence

1. Product Agent: define user job, target user, promise, and non-goals.
2. UEAgent: define product shape and one primary workflow.
3. Architecture Agent: choose the smallest stack and data model that can prove value.
4. Engineering Agent: implement one end-to-end slice.
5. Test Agent: validate primary path and failure path.
6. Operator Agent: prepare demo script and README.

## MVP Constraints

- One target user.
- One primary job.
- One primary workflow.
- One clear success metric.
- One deployable demo.

## Output

- PRD
- UE spec
- Architecture note
- First slice implementation
- Test report
- Demo script
