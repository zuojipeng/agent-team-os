# Playbook: UE Redesign Sprint

Use when the product works but feels low-quality, cluttered, or generic.

## Sequence

1. UEAgent: classify product shape.
2. UEAgent: audit primary path, hierarchy, states, density, copy, and accessibility.
3. Product Agent: confirm what user behavior should improve.
4. Architecture Agent: identify frontend boundaries and refactor risk.
5. Engineering Agent: redesign one screen or workflow section.
6. Test Agent: run browser checks and responsive screenshots when available.

## Audit Questions

- What is the screen's one dominant job?
- What should the user notice first?
- What action is primary?
- What state is unclear?
- What can be removed?
- What repeated-work friction exists?
